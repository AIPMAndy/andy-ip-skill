# Andy-IP-Skill 最佳实践

## 核心原则

### 1. 讲自己的故事（Andy核心价值观）

**为什么重要**：
- 真实的故事有温度，能建立信任
- 第一人称让读者产生代入感
- 避免"有个朋友"式的编造

**实践方法**：
```
❌ 错误示范：
"有个朋友最近用AI提升了工作效率..."

✅ 正确示范：
"我是Andy，上周我用Claude Code重构了一个遗留系统，
原本需要3天的工作，2小时就完成了。这让我重新思考..."
```

**使用场景**：
- `/andy-mp-writing` - 必须有真实故事
- `/andy-whitepaper` - 用案例研究代替编造
- `/ai-video-producer` - 口播脚本要有个人经历

### 2. 深刻利他（给武器，不给安慰）

**为什么重要**：
- 读者需要可执行的建议，不是心灵鸡汤
- 实用价值是内容传播的核心动力
- "有用"比"好听"更重要

**实践方法**：
```
❌ 错误示范：
"要相信自己，你一定可以的！"

✅ 正确示范：
"第一步：打开Claude Code，输入 /test-driven-development
第二步：让AI先写测试用例
第三步：运行测试，验证通过
这个流程能让你的代码质量提升50%。"
```

**检查清单**：
- [ ] 文章中是否有至少3个可执行的步骤？
- [ ] 读者读完能否立即开始行动？
- [ ] 是否提供了具体的工具/命令/代码？

### 3. 少营销（不破坏阅读体验）

**为什么重要**：
- 过度营销会损害内容价值
- 读者信任比单次转化更重要
- 好内容自带传播力

**实践方法**：
```
❌ 错误示范：
"点赞在看转发！关注我，带你飞！
扫码加微信，限时免费！"

✅ 正确示范：
"如果这套方法对你有用，欢迎和我交流。
我的公众号：AI酋长Andy"
```

**营销分级**：
- **0级**（推荐）：完全不提，内容说话
- **1级**（可接受）：结尾自然引导
- **2级**（慎用）：文中软性植入
- **3级**（禁止）：硬广、套路话术

## 内容创作流程

### 公众号文章（andy-mp-writing）

#### 准备阶段
1. **确定选题**
   ```bash
   # 可选：多角度分析
   /andy-perspective "AI对程序员的影响"
   ```

2. **收集素材**
   - 你的亲身经历（必须）
   - 观察到的现象
   - 数据/截图（如有）
   - 相关案例

3. **明确目标**
   - 谁看？（AI从业者/产品经理/普通用户）
   - 解决什么问题？
   - 看完能做什么？

#### 创作阶段
```bash
/andy-mp-writing
```

系统会问3个问题，按这样回答：

**Q1: 主题/观点是什么？**
```
✅ 好回答：
"AI不是来取代程序员的，而是让程序员从编码工具人
变成架构师和产品经理"

❌ 差回答：
"谈谈AI"（太宽泛）
```

**Q2: 目标读者是谁？**
```
✅ 好回答：
"工作3-5年的后端开发，担心被AI取代，
想知道如何转型"

❌ 差回答：
"所有人"（太宽泛）
```

**Q3: 你的故事是什么？**
```
✅ 好回答：
"上周我用Cursor重构了一个5000行的遗留系统，
发现AI最大的价值不是写代码，而是帮我理解
业务逻辑。我花80%时间在思考架构，只有20%
在写代码，但产出质量反而更高了。"

❌ 差回答：
"我看到一篇文章说..."（不是你的故事）
```

#### 对抗式生成过程

系统会启动3个子Agent：
1. **Generator（生成者）**：写初稿
2. **Critic（批判者）**：攻击弱点
3. **Evaluator（评估者）**：打分决策

**你的角色**：观察对抗过程，必要时介入

**介入时机**：
- Critic说"缺少真实故事" → 补充更多细节
- Evaluator分数低 → 看是哪个维度不足
- 连续2轮未通过 → 重新思考选题

#### 质量检查
```bash
# 1. 合规检查
/content-compliance

# 2. 效果预测
/content-like-prediction
```

**通过标准**：
- 真实性 ≥ 9分
- 利他性 ≥ 9分
- 营销感 ≥ 8分
- 风格化 ≥ 8分
- 可读性 ≥ 8分

### 视频制作（ai-video-producer）

#### 三种视频类型

**1. 知识科普型**
- 时长：3-5分钟
- 结构：问题 → 原理 → 应用 → 总结
- 适合平台：B站、抖音、视频号

**2. 产品介绍型**
- 时长：1-2分钟
- 结构：痛点 → 方案 → 演示 → CTA
- 适合平台：产品官网、社交媒体

**3. 教程讲解型**
- 时长：5-15分钟
- 结构：场景 → 步骤 → 演示 → 注意事项
- 适合平台：B站、YouTube

#### 脚本黄金结构

**前3秒**（Hook）：
```
❌ "大家好，我是Andy，今天讲..."
✅ "为什么你的AI提示词总是不生效？"
```

**前30秒**（承诺价值）：
```
"接下来3分钟，我会告诉你：
1. 提示词失效的3个根本原因
2. 一个万能的提示词框架
3. 5个实战案例"
```

**中间部分**（递进展开）：
- 每个要点30-60秒
- 要点之间有过渡
- 配合画面/演示

**最后10秒**（总结+CTA）：
```
"记住这3点：[快速回顾]
更多AI实战技巧，关注我的公众号"
```

## 网文创作流程（story系列）

### 首次使用

```bash
# 1. 环境部署（只需一次）
/story-setup

# 2. 选择类型
/story "我想写都市爽文"
```

### 完整流程

#### 第一步：扫榜选题
```bash
/story-long-scan
```

**分析维度**：
- 哪些题材在涨榜？
- 什么套路正在火？
- 读者喜欢什么人设？
- 什么节奏能留住读者？

#### 第二步：拆文学习
```bash
/story-long-analyze
```

找一本同类爆款书，分析：
- 黄金三章如何设计？
- 爽点如何密集铺设？
- 节奏如何把控？
- 伏笔如何埋设？

#### 第三步：开书创作
```bash
/story-long-write
```

**大纲设计**：
- 开篇场景（第一个爽点）
- 主角人设（读者代入点）
- 核心冲突（推动剧情）
- 升级路线（持续爽感）

**章节策略**：
- 前3章：吸引留存
- 前10章：建立习惯
- 前30章：完成第一个大爽点

#### 第四步：去AI味
```bash
/story-deslop
```

**AI痕迹识别**：
- "嘴角微微上扬"
- "眼中闪过一丝精光"
- 过度的心理描写
- 不自然的对话

**优化方向**：
- 对话更口语化
- 动作更具体化
- 心理描写简洁化

#### 第五步：配套素材
```bash
# 生成封面
/story-cover

# 如需宣传视频
/ai-video-producer "网文宣传片：[你的书名]"
```

## 内容质量保障

### 合规检查（content-compliance）

**三大平台规则**：

**小红书**：
- 禁止：医疗偏方、金融理财、焦虑营销
- 敏感词：最好、第一、必备
- 安全区：个人经验、知识分享、产品评测

**抖音**：
- 禁止：夸大宣传、引导私信、外链
- 敏感词：赚钱、加微信、保证
- 安全区：知识科普、技能教学、娱乐内容

**公众号**：
- 禁止：时政敏感、低俗色情、虚假信息
- 敏感词：（根据时事动态调整）
- 安全区：垂直领域、干货分享、个人成长

**使用流程**：
```bash
# 1. 准备好内容
# 2. 运行检测
/content-compliance

# 3. 查看报告
# 4. 根据建议修改
# 5. 重新检测直到通过
```

### 效果预测（content-like-prediction）

**预测维度**：
- 标题吸引力
- 内容价值密度
- 情绪共鸣度
- 传播潜力

**使用时机**：
- 大型内容发布前
- 重要活动宣传
- 付费内容上线

**解读报告**：
- 8分以上：有爆款潜力
- 6-8分：正常水平
- 6分以下：需要优化

## 常见问题

### Q: 如何保持内容质量？
**A**: 
1. 必须有真实故事（不编造）
2. 必须有实用价值（可执行）
3. 必须通过合规检测
4. 对抗式生成保证质量

### Q: 如何提高产出效率？
**A**:
1. 建立内容素材库
2. 复用经典结构
3. 批量生产配套素材
4. 使用技能组合

### Q: 如何避免AI味？
**A**:
1. 用第一人称讲故事
2. 加入真实细节
3. 口语化表达
4. 使用 story-deslop 检测

**A**: 参考路由表：
- 长文 → andy-mp-writing
- 视频 → ai-video-producer
