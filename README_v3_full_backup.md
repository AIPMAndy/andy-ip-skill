## 🏗️ 架构设计

### 三层智能架构

```
┌─────────────────────────────────────────────────────────┐
│           andy-ip-skill (智能路由层)                     │
│                                                          │
│  ▸ 新手教程模式 - 交互式引导，完成第一次实际使用          │
│  ▸ 任务前路由 - 根据需求智能匹配最佳工具                 │
│  ▸ 任务后导航 - 基于结果动态推荐下一步                   │
│                                                          │
│  触发方式：/andy-ip-skill 或 /andy-ip                    │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│              6大核心领域 (能力层)                         │
│                                                          │
│  🧬 基础设施  |  🎯 产品战略  |  ✍️ 内容创作             │
│  📱 平台发布  |  📊 数据研究  |  💻 开发工具             │
│                                                          │
│  每个领域包含多个专业工具，支持独立调用或组合使用          │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│          50+ 子项目 (执行层)                              │
│                                                          │
│  GitHub开源仓库 + 本地Claude Skills + 飞书工具集          │
│  所有工具开箱即用，支持自定义配置和二次开发                │
└─────────────────────────────────────────────────────────┘
```

### 设计原则

1. **单步决策，动态导航** - 不预设长流程，每步根据实际结果决定下一步
2. **Skills即产品** - 每个工具独立交付价值，可灵活组合
3. **智能路由优先** - 降低工具选择成本，让AI帮你找到最佳路径
4. **实战验证优先** - 所有工具经过Andy本人验证，确保真实产生价值

## 💼 典型工作流

### 场景1：创作公众号爆款文章（完整链路）

```bash
# Step 1: 召回相关记忆和经验
/andy-memory-loop recall "公众号创作"

# Step 2: 收集灵感素材
/story-import [粘贴素材内容]

# Step 3: 分析选题方向
/story-long-scan "AI+产品"领域

# Step 4: 创作文章（对抗式生成）
/andy-mp-writing "选定的主题"

# Step 5: 合规检查
/content-compliance wechat

# Step 6: 效果预测
/content-like-prediction

# Step 7: 发布到多平台
发布到公众号 + /lark-doc（备份到飞书）

# Step 8: 数据复盘和记忆沉淀
/story-long-analyze
/andy-memory-loop remember "本次创作的关键洞察"
```

**涉及工具**: andy-memory-loop, story-import, story-long-scan, andy-mp-writing, content-compliance, content-like-prediction, lark-doc
